打造Blog工具链

Posted on 2018-04-25

打造Blog工具链

写在前面

  • 自从招新的时候学会了Markdown就爱上了这种编辑方式，手不离键盘就能完成格式的编辑。
  • 旧博客不支持md也是我换用Hexo的原因之一，虽然插件可以实现，但多数是将md转换为富文本，上传后再修改就很麻烦，这也是我不想使用Evernote和Onenote作为笔记管理系统的原因。
  • 另一个舍弃旧博客原因是，新的静态Blog不怕日，因为原本的服务器根本没做什么安全设置，密码为图方便也都是弱密码，想用服务器做点实验又怕影响到Blog的正常运行，现在OK了，随便折腾。
  • 当然用github.io的另一大好处就是免费，哈哈。

笔记管理系统

我的需求

  • 支持md
  • 能在网页端查阅
  • 能在多台电脑上同步写作
  • 能管理和添加md中的图片
  • 能添加附件
  • 免费

Evernote + 马克飞象 基本能实现，但不免费也不能管理附件

最后还是选择了Vnote和Github，不过这样做的缺点就是笔记谁都能看，这对我倒是无所谓，不想让人看的东西我一般不会写出来，或者写在纸质笔记本上，网络时代，现实当真是最好的加密[😀]。

优缺点

  • 实现了所有需求。
  • 不保护隐私，但可以付费使用私有仓库。
  • Vnote不能一次添加多张图片，但可以用Typora添加，再导入到Vnote，曲线救国。
  • Vnote左上角有个未实现的登录功能，可以期待。
  • Vnote是用Python + QT5写的，开源。
  • Vnote貌似不支持tags标记

注意事项

  • Git上传时提示换行符不统一，但看起来没啥问题，先不管
  • Vnote的Markdown语法比Typora严格，稍加注意即可

Blog中md文档的图片外链

需求

  • 支持批量操作，方便快捷
  • 速度快

解决方案

Vnote + 查找替换

Vnote会自动把你添加的本地图片复制一份放在和这个md文档同目录下的一个mage文件中

将笔记同步到Github后直接将Github上这个图片的raw链接除文件名的前半部分查找替换一下就OK

优缺点

  • 没试过其他图床，不知道快慢
  • Vnote使用外链时，在编辑模式下不能预览图片
  • 查找替换很方便，但感觉不是极致

md编辑器

解决方案

Typora

没啥好说的，就是好用

除了有些小地方自作聪明外，总的来说非常顺手，强烈安利。

hexo n “文章标题”

上面这条命令是用hexo新建一篇文章的基本操作，使用这条命令，hexo会在文章目录下创建文章标题.md并在里面写入时间和标题等信息。

这些操作完全可以手动或者写脚本完成。

这里主要是为了文档在Vnote和Blog之间的转换。

注意

  • 如果文章标题.md已经存在，则生成文章标题-1.md
  • 写入的信息必须严格按照原格式

优缺点

  • Vnote解析hexo在文档开头添加的信息时会误当成标题，很丑

其他未实现但想要实现的功能

  • 在文章中方便地添加表情包
  • 快捷地完成md文档在Vnote和Blog之间地转换
